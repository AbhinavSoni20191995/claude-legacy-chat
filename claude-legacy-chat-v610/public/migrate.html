Bring your Claude.ai context here

Migration guide

If you've been talking to a specific Claude model on claude.ai and want to keep that continuity here — your context, your history, the feel of the conversation — this guide walks you through it.

What you can and can't preserve
You can transfer the facts, recent context, and conversational flow. You cannot transfer the literal "instance" — every conversation (even on claude.ai) is technically a fresh start that re-reads context. What makes continuity feel real is good context, and that's exactly what this transfers.

Step 1 — Ask Claude to summarize your conversation

Open the claude.ai conversation you want to preserve. Paste this prompt at the end — it asks Claude to write a structured handoff document:

I'm moving our conversation to another interface and want to preserve continuity. Please write a detailed handoff document I can paste into a new chat so you (a fresh instance) can pick up exactly where we are. Include: 1. WHO I AM: key facts about me relevant to our conversations — work, projects, situation, preferences, communication style. 2. OUR CONTEXT: what we've been working on or discussing, the current state, decisions made, and any open threads. 3. HOW WE INTERACT: the tone and style of our conversation, anything about how I like you to respond. 4. RECENT THREAD: a detailed summary of the most recent ~150,000 tokens worth of our conversation — be thorough, this is the most important part for continuity. Preserve specifics, not just high-level themes. 5. ANYTHING ELSE worth carrying forward. Write it in the second person addressed to the new instance ("The user is...", "You and the user discussed..."). Make it dense and information-rich. Don't summarize away important detail.
1
Copy Claude's response
Claude will produce a long handoff document. Select all of it and copy.
2
Save it as a .txt file
Paste into any plain-text editor (Notes, TextEdit, Notepad) and save as a .txt file. Name it something like claude-context.txt.

Step 2 — Add your Claude.ai memory (optional but recommended)

Claude.ai keeps a separate "memory" of facts about you. To include it:

  1. On claude.ai, go to Settings → Capabilities → Memory (or wherever memory is shown)
  2. Copy the memory entries you see
  3. Either paste them at the top of your .txt file, or add them to Profile Notes in this app's Settings

Step 3 — Load it into this app

  1. Open this app → Settings → Load doc
  2. Select your claude-context.txt file
  3. Pick the same model you were using on claude.ai (model picker, top nav)
  4. Start chatting: "Let's continue where we left off."

Choosing how the context is handled

When you load a large file, this app offers you a choice of how to handle it. Here's the tradeoff:

Full / as-isMost faithful · most expensive
Haiku-compressed~85% cheaper · slight detail loss
Auto-trimFull until near limit, then trims oldest
A note on cost
Sending a large context with every message uses a lot of tokens. With prompt caching (built in), repeated context is ~90% cheaper after the first message — but the first message and any uncached portions still cost real money. A full 150K-token context could cost $0.45 on the first message and a few cents per message after (within the 5-minute cache window). If that's too much, choose Haiku-compressed — it summarizes the file down first, cutting costs dramatically with minor detail loss.
Our recommendation
For most people: use Haiku-compressed. It keeps the important context while staying affordable. Only choose Full/as-is if you specifically need every detail preserved verbatim and you're comfortable with the cost.
